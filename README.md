# 次の一冊（静的サイト版）

直近に読んだ2冊から、パラメータ照合で次の1冊を見つけるツールです。
**サーバー不要・APIキー不要・完全無料**で動作します。

## 使い方（自分のPCで試す）

`index.html` をダブルクリックしてブラウザで開くだけです。
インストールや起動コマンドは一切不要です。

画面上部に「オンライン優先」「常にオフライン」の切り替えボタンがあります。
デフォルトは「オンライン優先」で、Google Books APIに接続できなかった場合のみ
自動的に内蔵データベースに切り替わります（バッジに理由が表示されます）。

接続できない場合は「→ オンライン接続を再試行」を押すと再接続を試みます。

### もし「内蔵データベースで動作中」のままになる場合

- バッジ下に表示される「理由」を確認してください
- `file://` から直接開いている場合、ブラウザによってはこの方式での外部API通信が
  制限されることがあります。その場合は以下のいずれかをお試しください：
  - 別のブラウザで試す
  - 簡易ローカルサーバー経由で開く:
    ```
    python3 -m http.server 8000
    ```
    その後ブラウザで `http://localhost:8000` を開く
  - 広告ブロッカーやセキュリティ系拡張機能を一時的に無効化してみる

## 仕組み

1. **Google Books API**（無料・登録不要）に直接アクセスし、毎回ランダムなジャンルから
   実在の書籍候補を取得します（数百万冊規模）。
2. 各候補のカテゴリ情報（Fiction / Poetry / Philosophy など）から、SF度・思想密度・
   感情強度など8つのパラメータをその場で推定します。
3. 入力した2冊のパラメータの中間地点に少しランダム性を加えた「目標値」を計算し、
   そこに最も近い本を選びます。
4. もし何らかの理由でGoogle Books APIに接続できない場合は、自動的に内蔵データベース
   （49冊）に切り替わります（画面上部のバッジで状態が分かります）。

選考アルゴリズムは意図的にブラックボックスにしてあります。

## 無料で公開する方法

### 方法1: GitHub Pages（推奨）

1. GitHubでリポジトリを新規作成
2. このフォルダの `index.html` をアップロード（リポジトリ直下に置く）
3. リポジトリの Settings → Pages → Source で「main」ブランチを選択して保存
4. 数分後、`https://あなたのユーザー名.github.io/リポジトリ名/` でアクセス可能に

### 方法2: Netlify / Vercel（もっと簡単）

1. https://app.netlify.com/drop にアクセス
2. `index.html` が入ったフォルダをそのままブラウザにドラッグ&ドロップ
3. 即座にURLが発行されて公開完了

どちらも無料プランで十分に使えます。

### HTTP 429（レート制限）が出る場合

Google Books APIはキーなしだと匿名アクセス扱いになり、IPごとの利用上限が
低く設定されています。429エラーが頻発する場合は、無料のAPIキーを設定してください。

**キーの取得方法（無料・5分程度）:**

1. https://console.cloud.google.com/apis/credentials にアクセス（Googleアカウントが必要）
2. プロジェクトを新規作成（または既存のものを選択）
3. 「APIとサービス」→「ライブラリ」で **Books API** を検索して有効化
4. 「認証情報」→「認証情報を作成」→「APIキー」でキーを発行
5. 発行されたキーを画面上の「Google Books APIキー」欄に貼り付けて保存

キーを設定すると利用上限が大幅に緩和され、429エラーがほぼ起きなくなります。

## 注意事項

- Google Books APIは無料枠（1日あたり1,000リクエスト程度）の範囲で使用しています。
  個人利用であれば問題になることはまずありません。
- パラメータ推定はあくまでカテゴリ情報からのヒューリスティックです。
  完璧な精度ではなく、「意外な発見」を楽しむための仕組みとして設計されています。
